Appendix Navigation Pattern

Information Architecture

The appendix has 5 sections (Journeys, Usability, What Changed, Consistency, Iterations) currently behind horizontal tabs. Only one is visible at a time. With all tabs needing a readability overhaul, the question is whether tabs are even the right pattern — or if a different navigation approach would help designers digest the information better.

Keep the horizontal tabs but improve them: add count badges showing how many items each tab has, improve tab naming (shorter, clearer), and fix the content layout within each tab. The structure stays the same — only one section visible at a time, switch via top tabs.

Works well when

  • Designers are already familiar with the current layout
  • Minimal implementation change — just improve content
  • Clean separation between unrelated sections

Watch out for

  • Still hides 4/5 sections behind clicks
  • Can't compare across sections (e.g., usability + consistency)

Replace tabs with vertical accordion sections. Each header shows the section title + a key metric (count or score). Click to expand/collapse. Multiple sections can be open simultaneously — so you can compare usability scores with consistency findings side by side. I recommend this because it lets designers see everything in context without switching views, and the headers give a scannable overview even when collapsed.

Works well when

  • Designers want to see multiple sections at once
  • Headers serve as a mini-summary even when collapsed
  • Natural vertical scroll — no hidden content
  • Key metrics visible at a glance on every header

Watch out for

  • All open = very long page
  • Less clear visual boundary between sections vs tabs

No navigation at all — all 5 sections render vertically, always visible. You scroll through them like a document. A sticky "jump to" floating nav lets you hop between sections. Like reading a research paper — everything is there, you just scroll.

Works well when

  • Everything discoverable — nothing hidden
  • Ctrl+F works across all sections
  • Natural reading flow top-to-bottom

Watch out for

  • Very long page — overwhelming if all sections have content
  • Empty sections add clutter (Consistency when no violations)
  • Hard to find specific section without the jump nav

A persistent left sidebar nav within the appendix area. Clicking a nav item scrolls the right content area to that section (or swaps content like tabs). The nav is always visible — you always know where you are and can jump instantly. Similar to documentation sites.

Works well when

  • Always-visible nav — no guessing what's available
  • Familiar docs-site pattern
  • Active state shows current position

Watch out for

  • Takes horizontal space from content
  • Still one-section-at-a-time (same as tabs, different chrome)
  • Nested sidebars (evidence viewer also has a sidebar) might confuse
