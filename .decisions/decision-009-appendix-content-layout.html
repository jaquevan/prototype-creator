Appendix Content Layout

Visual / UX

Within each tab, how is the information presented? Currently it's dense tables and text blocks. The redesign needs to make scores glanceable, findings scannable, and persona data digestible. This decision sets the pattern that all 5 tabs will follow.

Scores rendered as horizontal progress bars with color coding (green 2.5+, amber 1.5-2.4, red below 1.5). Below the scores, per-persona details are in expandable cards (collapsed by default). Click to reveal findings, confusion events, and think-aloud excerpts. I recommend this because it surfaces the key signal (scores) immediately while making detail available on demand.

Works well when

  • Scores are the primary signal — bars make comparison instant
  • Detail is available but not overwhelming
  • Same pattern works across Usability, Consistency, and Iterations

Watch out for

  • Expanding cards adds clicks for users who want all detail
  • Color-coded bars need accessible alternatives

Narrative + Highlights

Usability Summary
The prototype scored 15.5/21 across 7 usability dimensions. Workflow continuity and accessibility scored highest. Mental model fidelity scored lowest — scheduling terminology doesn't match how MLOps engineers think about queue states.
Key Issue
Column headers use "Pending/Admitted/Active" but users expect "Queued/Scheduled/Running"
Strength
Full keyboard navigation, visible focus indicators, ARIA labels present
AI-generated prose summary with inline highlighting. Key issues called out with colored left-border cards (red = problem, green = strength). Reads like a brief rather than raw data. Individual scores available in a collapsed "raw data" section below.

Works well when

  • Designer wants to READ, not parse — human-readable narrative
  • Key issues/strengths immediately obvious
  • Feels like a research report, not a dashboard

Watch out for

  • Prose generation varies — inconsistent quality
  • Raw scores hidden — harder to compare across runs
  • Longer to read than scanning a table or bars
